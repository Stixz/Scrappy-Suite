# **🔥 SCRAPPY STRUCTURE SUMMARY — The New Data Architecture**

## **1. “Tables” is dead.

Long live the Scrappy Grid.**

The old “Tables Module” is removed as a standalone module.

Iintroducing a **Scrappy Primitive**:

### **Scrappy Grid**

A lightweight, flexible, embeddable data surface.

Not a spreadsheet.  
Not a database.  
Not a module.

A **tool**.

---

# **2. Scrappy Grid is embeddable everywhere**

Any module can summon it:

- DirT Writer
- Notes
- Journal
- Canvas
- Future modules

This makes it feel like a **native building block**, not a separate app.

---

# **3. The Grid is a _foundational_ free feature**

Free tier includes:

- basic rows/columns
- drag‑to‑reorder
- simple sorting
- simple filtering
- inline embedding
- basic grid editing

This keeps Scrappy’s identity intact:

**personal, improvisational, artifact‑driven.**

Premium features (optional) sit _on top_ of this, not behind it.

---

# **4. Advanced “database-like” features become premium**

# DO NOT INCLUDE THIS SECTION NOW

Premium tier gets:

- multiple views (kanban, calendar, gallery)
- computed fields / formulas
- conditional formatting
- linked grids
- saved filters
- templates
- advanced export
- relations
- grid‑to‑canvas linking
- grid‑to‑timeline linking

This creates a **clean, ethical upgrade path** without locking creativity behind a paywall.

---

# **5. The Grid supports multiple _modes_ of structure**

Same component, different views:

- table
- checklist
- tracker
- roster
- content calendar
- worldbuilding matrix
- stat block
- character sheet
- budget scratchpad

Scrappy becomes a **creative artifact engine**, not a productivity clone.

---

# **6. The data model is JSON-based and Scrappy-native**

Every grid is stored as:

- rows
- columns
- cell types
- metadata
- layout state

This makes it:

- portable
- exportable
- safe
- easy to back up
- easy to restore
- easy to embed


---

# **7. The Grid can pop out into its own panel**

Not a module — just a **full-screen view** of the same component.

This keeps Scrappy panel‑agnostic and consistent.

---

# **8. The philosophy behind all this**

Scrappy is:

- chaos-friendly
- personal
- modular
- improvisational
- artifact-first

So:

- **Free tier = tools that help you create**
- **Premium tier = tools that help you scale**

This is the cleanest, most ethical, most sustainable model.

---

# **THE TL;DR VERSION**

**Remove the Tables Module.  
Replace it with the Scrappy Grid — a core primitive.  
Make basic grid functionality free.  
Make advanced database features premium.  
Embed the grid everywhere.  
Store it in a safe, simple JSON format.  
Let it pop out into its own panel when needed.**

This gives Scrappy:

- more power
- more flexibility
- more identity
- more future-proofing
- more trust from users

And it aligns perfectly with the studio’s ethos.
