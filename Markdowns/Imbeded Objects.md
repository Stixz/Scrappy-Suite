# **🧱 1. Text Blocks (already exist)**

DirT Writer already treats text as a flowing block system:

- paragraphs
- headings
- lists
- quotes
- inline formatting

This is the baseline.

---

# **🖼️ 2. Image Blocks (Scrappy-native images inside text)**

Yes — this means:

### **You can insert an image _mid‑text_ inside DirT Writer.**

Not as an attachment.  
Not as a separate module.  
Not as a “media panel.”

A **block**.

Like:

```
[paragraph]
[paragraph]
[image block]
[paragraph]
```

This unlocks:

- character portraits
- item art
- moodboards
- diagrams
- UI mockups
- worldbuilding visuals
- reference images
- concept art
- maps

And because it’s a block, you should be able to:

- drag it
- resize it
- caption it
- align it
- embed metadata

This is the same philosophy as Notion, Craft, Obsidian Canvas — but **Scrappy‑flavored**.

---

# **💻 3. Code Blocks (inline or block-level)**

Again, yes — this means:

### **You can insert code directly inside DirT Writer.**

Two modes:

### **Inline code**

For quick snippets:

```
Use `myFunction()` to initialize the module.
```

### **Block code**

For full sections:

````markdown
```typescript
function spawnFogre() {
  return new ChaosEntity("Fogre");
}
```
````

This is huge for:

- Scrappy plugin development
- system design notes
- technical documentation
- JSON structures
- pseudo-code
- logic diagrams

DirT Writer becomes a **hybrid creative + technical surface**, which fits your brain perfectly.

---

# **🧩 4. Why these are “Scrappy Primitives”**

Because they’re not modules.  
They’re not apps.  
They’re not features hidden behind menus.

They’re **building blocks**.

Just like:

- text
- images
- code
- grids (the new one)
- canvas nodes
- embeds

Scrappy becomes a **block-based creative engine**, not a monolithic editor.

---

DirT Writer becomes a **true artifact builder**, not just a text editor.
